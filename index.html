Šta je Arduino?

Arduino je mikrokontrolerska platforma razvijena 2005. godine na Institutu za interaktivni dizajn u Ivreji, Italija. Open-source hardverska i softverska platforma koja omogućava upravljanje elektronskim komponentama kroz C/C++.

Platforma se zasniva na 8-bitnom Atmel AVR mikrokontroleru (popularni Uno) ili 32-bitnom ARM procesoru (napredniji Due). Programira se putem besplatnog Arduino IDE softvera (www.arduino.cc) koji radi na Windows, macOS i Linux sistemima.

Hardverska arhitektura

  • Digitalni I/O — HIGH (5V) ili LOW (0V)
  • Analogni ulazi — 0–5V → vrednosti 0–1023 (10-bit ADC)
  • PWM pinovi (~) — analogni izlaz digitalnim signalom
  • USB — komunikacija i upload programa
  • Napajanje — USB 5V ili eksterno 7–12V DC

Shield-ovi

Pločice koje se postavljaju direktno na Arduino i proširuju funkcionalnost bez lemljenja. Postoje shield-ovi za: WiFi, Bluetooth, Ethernet, motore, GPS, OLED i mnoge druge namene.

Sadržaj

LED dioda

Duža nožica = anoda (+), kraća = katoda (−). Otpornik 220Ω redno obavezno — bez njega dioda se uništava. Anoda → 220Ω → pin; Katoda → GND.

Taster

Pull-down 10kΩ između signal pina i GND. Otpušteno = LOW (0V), pritisnuto = HIGH (5V). Bez pull-down otpornika pin "pluta" i daje nepouzdana očitavanja.

Zujalica

Crvena/duža nožica (+) → digitalni pin (OUTPUT). Crna (−) → GND. Zvuk: tone(pin, frekvencija).

Ultrazvučni senzor HC-SR04

PinVeza
Vcc+5V
GNDGND
TrigDigitalni pin (OUTPUT)
EchoDigitalni pin (INPUT)

Foto-otpornik (LDR)

Analogni pin + pull-up 10kΩ ka +5V. Vrednosti: 0 (svetlo) → 1023 (mrak).

Servo motor

Crvena→5V, crna/braon→GND, žuta/narandžasta→PWM pin (~). Rotacija 0–180°. Biblioteka: Servo.h.

⚠️ Snažniji servo zahteva eksterno napajanje — spojiti masu eksternog napajanja sa GND Arduina!

RGB LED

4 nožice: R, G, B + zajednička katoda (→GND). Obavezno PWM pinovi. Uno PWM pinovi: 3, 5, 6, 9, 10, 11.

DHT11 / DHT22

Pin1→5V, Pin2→signal + pull-up 10kΩ, Pin3→slobodan, Pin4→GND. Biblioteka: DHT.h.

Relej

Vcc→5V, GND→GND, In→signal pin. NO (normalno otvoren): HIGH = uključen. NC (normalno zatvoren): inverzna logika.

LED matrica MAX7219

VCC→5V, GND→GND, DIN→pin12, CS→pin10, CLK→pin11. Biblioteka: LEDControlMS.h.

IR prijemnik

Leva nožica→digitalni pin, srednja→GND, desna→+5V. Biblioteka: IRremote.h. Prima 38kHz signale.

Princip PWM

Arduino digitalni izlazi daju samo 5V ili 0V. Za dimovanje LED ili upravljanje servo motorom potreban je napon između tih vrednosti. PWM brzo uključuje i isključuje signal simulirajući analogni napon. Ključni pojam: duty cycle — procenat vremena u kom je signal HIGH.

analogWriteDuty cycleEfektivni napon
00%0V
6425%~1.25V
12850%~2.5V
19275%~3.75V
255100%5V
PWM pinovi Uno: 3, 5, 6, 9, 10, 11 (označeni ~ na ploči). Mega: 2–13 i 44–46.
arduino / c++Fade + RGB mešanje boja
// Fade — postepeno dimovanje LED na pin 9
void loop() {
  for (int i=0; i<=255; i++) { analogWrite(9,i); delay(8); }
  for (int i=255; i>=0; i--) { analogWrite(9,i); delay(8); }
}

// RGB LED — R=9, G=10, B=11 (zajednička katoda)
// Žuta = R MAX + G MAX
analogWrite(9,255); analogWrite(10,255); analogWrite(11,0);
// Ljubičasta
analogWrite(9,180); analogWrite(10,0);   analogWrite(11,200);
// Bela = sve MAX
analogWrite(9,255); analogWrite(10,255); analogWrite(11,255);

Semafor
3 LED diode u semaforskom redosledu. Uvodni primer za digitalWrite() i delay().

Opis i kod

Tri LED diode (crvena pin 13, žuta pin 12, zelena pin 11) naizmenično svetle u pravom semaforskom redosledu. Svaka LED ima otpornik 220Ω ka GND. Odličan uvodni primer — demonstrira pinMode(), digitalWrite() i delay().

arduino / c++semafor.ino
// Semafor — crvena=pin13, žuta=pin12, zelena=pin11
void setup() {
  pinMode(11, OUTPUT); // zelena
  pinMode(12, OUTPUT); // žuta
  pinMode(13, OUTPUT); // crvena
}
void loop() {
  digitalWrite(11, HIGH); delay(3000); digitalWrite(11, LOW); // zelena 3s
  digitalWrite(12, HIGH); delay(1000); digitalWrite(12, LOW); // žuta 1s
  digitalWrite(13, HIGH); delay(3000); digitalWrite(13, LOW); // crvena 3s
  // crvena + žuta (prelaz nazad na zelenu)
  digitalWrite(13, HIGH); digitalWrite(12, HIGH);
  delay(1000);
  digitalWrite(13, LOW);  digitalWrite(12, LOW);
}

  for (int i=0; i<8; i++) pinMode(pinovi[i], OUTPUT);

  digitalWrite(13, v < prag ? HIGH : LOW);

Opis

Potenciometar na analognom ulazu A0 daje vrednosti 0–1023. Ove vrednosti se konvertuju u opseg 0–255 za PWM izlaz (analogWrite) koji kontroliše osvetljenost LED diode na pinu 9.

int ledPin  = 9;   // PWM pin (~)
int potPin  = A0;
int potValue, pinValue;

void setup() {
  pinMode(ledPin, OUTPUT);
  pinMode(potPin,  INPUT);
  Serial.begin(9600);
}

void loop() {
  potValue = analogRead(potPin);
  Serial.println(potValue);       // praćenje u Serial Monitoru

  // Konverzija 0–1023 → 0–255
  pinValue = (255.0 / 1023.0) * potValue;
  // Ili: pinValue = map(potValue, 0, 1023, 0, 255);

  analogWrite(ledPin, pinValue);
  delay(50);
}

Termistor — vrste

VrstaPonašanje otporaPrimena

PTC (Positive Temp. Coeff.)Otpor raste kad temp. rasteZaštita kola, strujna ograničenja

NTC termistor se koristi u naponskom deliocu: +5V → [R_termistor] → ● → [R1=10kΩ] → GND. Mikrokontroler meri napon u tački ●.

💡 Što je temperatura veća → otpor NTC manji → napon u tački ● veći → ADC vrednost veća.

Prednosti i nedostaci

✅ Visoka osetljivost, mala cena, brza reakcija, jednostavno povezivanje

Kod
arduino / c++termistor.ino — linearizovana aproksimacija
#include <math.h>
const float R1       = 10000.0; // fiksni otpornik 10kΩ
const float Vcc      = 5.0;
const float B_CONST  = 3950.0;  // B koeficijent termistora
const float R0       = 10000.0; // otpor na 25°C
const float T0       = 298.15; // 25°C u kelvinima

void setup() { Serial.begin(9600); }

void loop() {
  int adcVal = analogRead(A0);         // 0–1023
  float Vout = adcVal * (Vcc / 1023.0); // napon na tački ●
  float Rterm = R1 * (Vcc - Vout) / Vout; // otpor termistora

  // Steinhart-Hart aproksimacija (B parametar metoda)
  float tempK = 1.0 / (1.0/T0 + log(Rterm/R0)/B_CONST);
  float tempC = tempK - 273.15;

  Serial.print("Temp: "); Serial.print(tempC, 1); Serial.println(" °C");
  delay(1000);
}

Šta je galvansko odvajanje?

Galvansko odvajanje znači da između dva električna kola ne postoji direktna provodna veza — informacija se prenosi drugim putem. Kod optokapler-a to je svetlost (IR LED → fototransistor).

Koristi se kada: mikrokontroler treba zaštititi od visokog napona, ulazni signal dolazi sa drugog napona (npr. 24V industrijski), ili postoje smetnje i šum.

Princip rada

Na ulaz optokapler-a se dovede napon → LED unutar zasvetli
Svetlost aktivira fototransistor na izlaznoj strani
Fototransistor provodi → logička "1" na ulazu mikrokontrolera
Kad LED ne svetli → fototransistor isključen → logička "0"
⚠️ Mase ulaznog (24V) i izlaznog (5V Arduino) kola NE SME biti spojene — to bi poništilo galvansko odvajanje!

IoT-01Elektronika
Naponski delilac — izbor vrednosti otpornika
Zašto 10kΩ a ne 1kΩ? Struja, potrošnja, opterećenje, stabilnost, šum. Zlatna sredina za Arduino ADC.
+
Sadržaj

Osnovna formula

V_out = V_in × R2 / (R1 + R2) — odnos otpornosti određuje izlazni napon.

💡 Sama formula nije zavisna od veličine otpornika — ali veličina utiče na struju, potrošnju i stabilnost!

1. Struja kroz razdelnik

Sa 9V i 1kΩ+800Ω: I = 9V / 1800Ω = 5mA → P = 45mW — stalno gubi energiju!

Sa 10× većim otpornicima: I = 0.5mA → P = 4.5mW — 10× manja potrošnja, isti odnos napona.

2. Opterećenje (Arduino A0 pin)

Analogni ulaz Arduino ima ulaznu impedansu ~100MΩ — gotovo ne vuče struju, pa su 10kΩ–100kΩ sasvim dovoljni.

3. Stabilnost i šum

Previše mali otpornici (100Ω–1kΩ): troše previše struje, greju se. Previše veliki (>1MΩ): signal postaje "šumovit" jer mala smetnja ima velik efekat. Zlatna sredina: 10kΩ–100kΩ.

Proračuni za naponski delilac

Izračunavanje R1 za zadati izlazni napon: R1 = R2 × (Vin/Vout − 1)

Primer: 9V → 5V za A0, R2=10kΩ: R1 = 10k × (9/5 − 1) = 10k × 0.8 = 8kΩ (koristiti 8.2kΩ E24)

Struja kroz delilac: I = Vin / (R1+R2) = 9/(8k+10k) = 0.5mA. Disipacija: P = 9V × 0.5mA = 4.5mW — mala.

💡 Kondenzator 100nF paralelno sa R2 filtrira visokofrekventni šum na ADC ulazu. Za spore signale (temperatura, svetlost) sasvim dovoljan.

Problem — "lebdeće stanje"

Digitalni ulaz mikrokontrolera mora uvek imati definisan napon: HIGH ili LOW. Ako pin "lebdi" u vazduhu (nema ni HIGH ni LOW), prima slučajne smetnje i daje nepouzdana očitavanja.

Zašto 10kΩ?

Kada je dugme pritisnuto, otpornik "spaja" napajanje na masu (ili obratno). Struja: I = 5V / 10kΩ = 0.5mA. Disipacija: P = 5V × 0.5mA = 2.5mW — mala potrošnja, otpornik se ne greje. Dovoljno mali da drži stabilan logički nivo i otporan na smetnje. Preporučeni opseg: 4.7kΩ–20kΩ.

💡 Arduino ima i ugrađene interne pull-up otpornike (~20–50kΩ). Aktiviraju se sa: pinMode(pin, INPUT_PULLUP). Tada neaktivan taster = HIGH (obrnuta logika od pull-down!).

Arduino Uno ima 10-bitni ADC na pinovima A0–A5. Mereni napon (0–5V) se pretvara u broj 0–1023. Formula: V = analogRead(pin) × (5.0 / 1023.0).

Teorija

Princip rada — elektroluminiscencija

LED je dioda koja pri forward polarizaciji emituje fotone — elektroluminiscencija. Boja zavisi od band-gap materijala — energetskog procepa poluprovodnika koji određuje energiju (i talasnu dužinu) emitovanog fotona.

Kratka istorija: 1927. Oleg Losev (SiC LED); 1962. Nick Holonyak (prva crvena LED); rani 1990s Nakamura, Amano, Akasaki (plava LED → bela rasveta) → Nobelova nagrada 2014.

Primer A — 5V, crvena (Vf=2.0V), 15mA

Primer B — 9V, crvena (Vf=1.6V), 20mA

proračun

7 koraka univerzalnog rešavanja IoT problema

Korak 1 — Razumevanje problema: Šta sistem treba da radi? Koji je cilj? Da li reaguje na okolinu ili korisnika? Bez jasnog cilja nema dobrog IoT rešenja.
Korak 2 — Identifikacija ulaza i izlaza: Šta sistem "posmatra"? (senzori, tasteri, podaci) Šta sistem "menja" u okolini? (aktuatori, signali, obaveštenja)
Korak 3 — Apstrakcija (modelovanje): Realni svet → uprošćen model. Temperatura → broj; svetlo → ADC vrednost; uređaj ON/OFF → logičko stanje. Učenik radi sa modelom, ne sa stvarnošću direktno.
Korak 4 — Logička pravila odlučivanja: IF-ELSE, poređenja, pragovi, stanja. Ovo je "srce" IoT sistema — pamet sistema.
Korak 5 — Tehnička realizacija: Tek sada se bira mikrokontroler, senzori, aktuatori, način upravljanja. Tehnologija se bira na osnovu modela, ne obrnuto!
Korak 6 — Bezbednost i pouzdanost: Šta ako senzor ne radi? Preveliki napon? Da li je sistem bezbedan za korisnika? Električna zaštita.
Korak 7 — Testiranje i korekcija: Testiranje u realnim uslovima, podešavanje pragova, optimizacija. Rešenje nije "gotovo" kada se prvi put pokrene.
